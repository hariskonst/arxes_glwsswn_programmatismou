# myHTML Parser (Flex & Bison)

This project implements a **Lexical and Syntactic Analyzer** for the `myHTML` pseudo-language. [cite\_start]It was developed as part of the "Principles of Programming Languages & Compilers" course (Spring 2025) at the Department of Computer Engineering and Informatics (CEID), University of Patras[cite: 216, 217, 218].

## Project Overview

The tool takes a source file written in `myHTML` (a simplified version of HTML) and validates its syntax using a parser generated by **Flex** and **Bison**.

### Features

  * **Lexical Analysis:** Recognizes tokens, tags, and attributes using Flex.
  * **Syntactic Analysis:** Validates the structure of the code based on the defined BNF grammar.
  * **Attribute Handling:** Supports attributes in arbitrary order for tags like `<img>`, `<a>`, `<p>`, `<div>`, and inputs (implemented via Bison rules).
  * **Structure Validation:** Ensures correct nesting of elements like `<head>`, `<body>`, `<form>`, etc..

## Supported Tags

The parser currently supports the following `myHTML` elements:

  * **Structure:** `<MYHTML>`, `<head>`, `<body>`
  * **Metadata:** `<title>`, `<meta>`
  * **Content:** `<p>`, `<div>`, `<a>` (links), `<img>` (images)
  * **Forms:** `<form>`, `<input>`, `<label>`

## How to Compile & Run (Windows)

### Prerequisites

You need **Flex**, **Bison**, and **GCC** installed on Windows (e.g., via WinFlexBison or MinGW).

### Compilation Steps

Open your terminal (PowerShell or CMD) in the project folder and run:

```powershell
# 1. Generate the lexer C file
flex lexer.l

# 2. Generate the parser C and Header files
bison -d parser.y

# 3. Compile everything together
gcc lex.yy.c parser.tab.c -o myParser.exe
```

### Usage

[cite\_start]To parse a file, pass it as an argument to the executable[cite: 330]:

```powershell
.\myParser.exe test.txt
```

If the syntax is correct, the program will output the structure and a success message. [cite\_start]If there are errors, it will report the line number where the error occurred[cite: 196].

## File Structure

  * [cite\_start]`lexer.l`: Flex file containing the token definitions and regular expressions[cite: 198].
  * [cite\_start]`parser.y`: Bison file containing the BNF grammar and syntax rules[cite: 3].
  * [cite\_start]`test.txt`: An example input file for testing the parser[cite: 1].
